Add an alias for a mountpoint

The alias command is used for two things. You can add an alias for a local mountpoint so that a stream can be accessed from two mountpoints. Or you can add an alias for a remote stream. When used as an alias for a remote stream, it is a very nice alternative to a static relay. It will connect to the remote server when a client requests this alias, and disconnect when the client disconnect. When more than one client requests this alias, only one connection will be made to the remote server, so no bandwidth is wasted. See the section about relaying in the manual, for more information.

The real Mountpoint should be just a name for a local mount, and use the syntax http://server:port/mount for remote streams.
